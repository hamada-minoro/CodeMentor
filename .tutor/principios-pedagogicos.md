# Princípios Pedagógicos

Estes são os princípios que sustentam todo o desenho deste repositório. Toda IA, ao gerar conteúdo novo (aulas, exercícios, agents, skills), deve respeitar estes princípios.

1. **Fundamentos antes de frameworks.** Frameworks mudam. Lógica, tipos, estrutura de dados, testes e arquitetura permanecem.
2. **IA deve explicar, orientar, revisar e provocar raciocínio** — não apenas entregar respostas.
3. **IA não deve resolver exercícios pelo aluno na primeira tentativa.**
4. **O aluno deve tentar manualmente antes de pedir solução.** O erro faz parte do aprendizado.
5. **Toda aula deve gerar teoria, exercícios, pasta de respostas, revisão, nota e consolidação** — o ciclo completo descrito em `fluxo-de-estudo.md`.
6. **Python primeiro, sempre.** Dos Módulos 00 a 06, todo o conteúdo é só em Python, para o aluno separar "ideia de programação" de "sintaxe de uma linguagem" sem o peso de aprender duas linguagens ao mesmo tempo. TypeScript e Node.js entram como módulo dedicado (Módulo 07), e só a partir daí o conceito é comparado nas duas linguagens quando fizer sentido.
7. **O aprendizado deve ser incremental.** Cada módulo depende do anterior.
8. **O repositório deve funcionar com IA local e também com ferramentas pagas**, sem favorecer nenhuma.
9. **Recomendações sobre ferramentas pagas devem ser escritas com cautela**, avisando que preços e limites mudam.
10. **Linguagem clara, honesta e prática.** Sem hype, sem promessa de atalho de carreira.

## Por que isso importa

Código gerado por IA está cada vez mais acessível e rápido. O que diferencia um programador competente não é mais "saber digitar código", mas saber:

- entender o problema antes da solução;
- avaliar se o código gerado é bom ou ruim;
- escrever especificações claras;
- revisar arquitetura, testes e segurança;
- usar IA como ferramenta de produtividade, não como muleta.

Este repositório existe para formar esse tipo de programador — não um "operador de prompt frágil".
