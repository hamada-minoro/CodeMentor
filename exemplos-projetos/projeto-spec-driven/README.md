# Projeto Spec-Driven

Exemplo de projeto que demonstra o fluxo completo de Spec-Driven Development (veja `docs/GUIA_SPEC_DRIVEN_DEVELOPER.md` e `.tutor/curriculo/modulo-12-ia-spec-driven.md`).

## Objetivo

Mostrar maturidade com IA, documentação e planejamento — não apenas código funcionando.

## O que este projeto deveria ter

```text
docs/specs/001-feature/
  spec.md
  plan.md
  tasks.md
  tests.md
  review.md
CLAUDE.md
.claude/
README.md
```

## Exemplo incluído

A pasta [`docs/specs/001-exemplo-feature/`](docs/specs/001-exemplo-feature/) contém um exemplo completo de spec, plan, tasks, tests e review para uma feature fictícia ("lista de favoritos"), mostrando o nível de detalhe esperado em cada artefato.

Este exemplo é **apenas documentação** — não há implementação de código associada, propositalmente. O objetivo é que você use esse exemplo como referência de formato ao criar a spec da sua própria feature, em um projeto real (por exemplo, evoluindo a API financeira do Módulo 08/09).
