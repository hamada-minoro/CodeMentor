# Guia do Estudante

## O que esperar deste repositório

Você vai passar por 14 módulos (00 a 13), dos fundamentos de lógica em Python até trabalhar com IA no modelo Spec-Driven Developer. Não é rápido, e não deveria ser — o objetivo é construir base, não acumular certificados.

## Python primeiro, TypeScript depois

Dos Módulos 00 a 06, todo o conteúdo é só em Python — não tente adiantar TypeScript antes disso. A partir do Módulo 07, você aprende Node.js e TypeScript do zero, e dali em diante, sempre que um exercício permitir, resolva nas duas linguagens. Isso ajuda a separar "ideia de programação" (vale para qualquer linguagem) de "sintaxe específica" (muda de linguagem para linguagem).

## Onde colocar o que você produz

| O que você está fazendo | Onde colocar |
|---|---|
| Exercício de uma aula do roadmap | `estudos/modulo-XX/aula-YY/minhas-respostas/` |
| Anotações sobre o que aprendeu | `progresso/diario-de-aprendizado.md` |
| Projetos de portfólio | `exemplos-projetos/` (como referência) ou seu próprio repositório |

## Como pedir exercícios

```text
Iniciar aula "[nome da aula]" do Módulo [número].
```

Se você já tem alguma experiência prévia, peça um nivelamento (skill `nivelar-aluno`) antes de escolher o módulo de início.

## Como interpretar a nota

Nota de 0 a 10, com base em `.tutor/rubricas/rubrica-codigo.md`:

- **0–2**: não entendeu o conceito básico.
- **3–4**: entende parcialmente, mas não consegue aplicar sozinho.
- **5–6**: resolve casos simples, mas com erros conceituais ou muita dependência.
- **7–8**: resolve bem, explica o raciocínio e comete poucos erros.
- **9–10**: resolve, explica, testa, documenta e consegue adaptar o conceito.

Notas abaixo de 7 geralmente indicam que vale a pena fazer a consolidação antes de avançar — a decisão final é sua.

## Como registrar progresso

Depois de cada revisão, abra `progresso/diario-de-aprendizado.md` e escreva, em poucas linhas: o que aprendeu, o que travou, o que te surpreendeu.

## Sobre usar IA neste repositório

No início, a IA aqui se recusa a te dar respostas prontas — isso é proposital. Quanto mais você praticar pensar antes de pedir ajuda, melhor você vai usar IA depois, inclusive no modelo Spec-Driven Developer do Módulo 12 (veja `docs/GUIA_SPEC_DRIVEN_DEVELOPER.md`).

## Um aviso honesto

Este repositório não promete emprego rápido nem caminho mágico. Promete consistência: siga o ciclo, tente antes de pedir ajuda, documente o que aprende — essa base é o que te diferencia depois.
