# Rubrica de Spec-Driven Development

Use esta rubrica a partir da Fase 11, ao avaliar o trabalho do aluno com `spec.md`, `plan.md`, `tasks.md`, `tests.md` e `review.md`.

| Critério | Pontos |
|---|---|
| Spec tem critérios de aceitação verificáveis | 2 |
| Spec mapeia regras de negócio e casos de erro | 2 |
| Plan tem abordagem técnica coerente com a spec | 2 |
| Tasks são pequenas e verificáveis individualmente | 2 |
| Implementação seguiu uma task por vez, sem pular etapas | 1 |
| Review classifica achados como bloqueante/importante/melhoria/dúvida | 1 |

## Observações

- Penalize specs vagas ("o sistema deve funcionar bem") sem critério objetivo de aceitação.
- Penalize implementações que pularam direto para o código sem plan ou tasks.
- Valorize quando o aluno identificou ambiguidades na própria spec antes de implementar.
