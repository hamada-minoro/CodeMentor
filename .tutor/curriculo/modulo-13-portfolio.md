# Módulo 13 — Portfólio e projetos finais

**Duração sugerida:** contínua, conforme o aluno avança

## Objetivo do módulo

Consolidar o aprendizado em projetos publicados que mostrem raciocínio — não apenas tela bonita.

## Pré-requisitos

Módulos 00 a 12 concluídos.

## Projetos recomendados

1. **CLI de tarefas** — em Python e TypeScript, mostrando lógica, manipulação de arquivos e estrutura de dados.
2. **API financeira** — Node.js + TypeScript + Fastify/NestJS + PostgreSQL + Prisma, mostrando backend real, autenticação, testes e documentação.
3. **Automação com Python** — resolver um problema real com scripts (organizar arquivos, ler CSV e gerar relatório, consumir API pública, gerar PDF simples, scraping permitido de dados públicos).
4. **Sistema com Spec-Driven Development** — projeto com `docs/specs/001-feature/` completo (`spec.md`, `plan.md`, `tasks.md`, `tests.md`, `review.md`), `CLAUDE.md` e `.claude/`, mostrando maturidade com IA, documentação e planejamento.
5. **Projeto final** — escolher um domínio real (controle de estoque, agendamentos, gestão de EPIs, CRM simples, helpdesk, etc.) com autenticação, banco de dados, testes, logs, Docker, documentação, deploy, specs, tasks e revisão técnica.

## Atividades / exercícios esperados

Publicar pelo menos 3 dos 5 projetos acima em repositórios próprios no GitHub, cada um com README completo explicando o problema, a solução, como rodar e as decisões técnicas tomadas.

## Critérios para avançar (checklist de maturidade — Nível 4)

- [ ] Entende camadas e separa regra de negócio de infraestrutura.
- [ ] Revisa código gerado por IA com critério.
- [ ] Escreve specs e quebra features em tasks.
- [ ] Sabe justificar decisões técnicas.
- [ ] Entrega ao menos um projeto publicado e documentado.

## Como a IA deve ajudar neste módulo

- Revisar READMEs e documentação dos projetos do portfólio, sugerindo o que falta para ficar claro a um leitor externo.
- Ajudar o aluno a escolher um foco estratégico para os próximos passos (backend, dados e automação, IA aplicada a software, ou arquitetura) — veja `docs/ROADMAP.md`, seção "Depois da base".

## Como a IA NÃO deve ajudar neste módulo

- Não construa o projeto final pelo aluno. Este módulo existe justamente para o aluno demonstrar autonomia.
