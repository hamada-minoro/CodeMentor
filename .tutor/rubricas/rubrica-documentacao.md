# Rubrica de Documentação

Use esta rubrica para avaliar READMEs e documentação de projetos (a partir do Módulo 02, quando projetos passam a ter README próprio).

| Critério | Pontos |
|---|---|
| Explica como rodar o projeto | 2 |
| Explica as decisões técnicas tomadas | 2 |
| Explica erros encontrados e como foram resolvidos | 2 |
| Documenta endpoints/contratos (quando aplicável) | 2 |
| Linguagem clara, sem jargão não explicado | 1 |
| Estrutura legível (títulos, listas, exemplos) | 1 |

## Observações

- Documentação não precisa ser longa para ser boa — precisa ser útil para alguém que nunca viu o projeto.
- Penalize promessas vagas ("funciona perfeitamente") sem explicação de como ou por quê.
